## 学习内容

### 🆕 阶段二：Web3 基础 — Network

> Handbook 原话：*"Web3 里的'网络'不是一个抽象背景，而是交易能否被打包、状态能否被同步、费用如何产生、确认需要多久、L2 和主网如何分工的基础环境。"*

#### 第一性原理

区块链网络的核心不是"存数据"，而是**让互不信任的参与者对状态变化达成一致**。

对比你熟悉的场景：
- 普通应用：服务器直接更新数据库 → 快，但你要信任服务器
- 链上系统：交易→节点传播→执行→打包→验证→确认 → 慢，但公开可验证

三个核心认知：
- **区块是同步节奏**：状态不是每秒连续更新，是按区块批量推进（像火车按班次发车）
- **共识是信任来源**：你相信状态不是因为某家公司说了算，而是网络按规则达成一致
- **网络选择改变一切**：主网、测试网、L2 的费用/速度/安全假设完全不同

#### 知识节点

- [ ] **Block**（初级）
  - 区块 = 交易被批量提交和排序的单位
  - 关键理解三点：交易有顺序 → 顺序影响结果 / 区块有 gas limit → 吞吐有限 / 新区块引用前一区块 → 形成可验证历史
  - 对比你已知的：区块 ≈ batch，但不是中心化批量，是去中心化批量

- [ ] **Consensus**（中级）
  - 共识 = 网络决定"哪段历史有效"的机制
  - 对开发者的实际影响：交易要等几个 confirmation 才安全 / 区块可能短暂重组 / RPC 异常时状态读取可能延迟
  - 类比：共识 ≈ 分布式系统的"最终一致"，但通过经济激励和密码学保证

- [ ] **PoS**（中级）
  - Proof of Stake = 验证者质押 ETH 参与区块提议，行为错误可能被惩罚
  - 解释了为何有"验证者""slashing""最终性"这些概念
  - 对普通用户：你不一定需要运行验证者，但要知道网络安全不是"免费"的

- [ ] **Testnet**（初级）
  - 测试网 = 在接近真实链的环境测试的一切
  - ⚠️ 测试网不能替代主网安全审查（流动性/MEV/攻击动机/资产规模都不同）
  - 做项目记录清单：部署哪条测试网、合约地址、RPC、chain id、测试币来源

- [ ] **L2**（中级）
  - L2 = 在主网安全基础上做扩展：交易在 L2 执行 → 结果提交回主网
  - 优势：费用低、确认快；代价：多了桥、提现等待、排序器复杂度
  - 产品设计注意：支持多 L2 时要展示当前网络、资产在哪、桥接时间

- [ ] **Rollup**（高级 — 了解即可）
  - 主流 L2 扩展路线：Optimistic Rollup / ZK Rollup
  - 关键判断：Rollup 降低了单笔成本，但没有消除链上系统复杂度

#### 在 AI × Web3 中的位置 🎯

这和你的 AI 学习**直接相关**：

> AI Agent 如果要读取链上状态或执行交易，必须知道自己在哪条网络上操作。主网和测试网、L1 和 L2、不同 chain id、不同合约地址，不能靠模型"猜"。

更稳妥的做法：让工具返回结构化信息（chain id / RPC / 区块高度 / 交易哈希 / 确认数 / explorer 链接），Agent 的总结应该引用可验证信息，而不是只说"交易成功了"。

→ 这就是你以后写 Agent Tool 时的网络层设计原则。
