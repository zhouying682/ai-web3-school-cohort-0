## 学习内容

### Prompt 章节核心要点

- [ ] **System Prompt vs User Prompt**: System Prompt 定义角色/边界/规则，User Prompt 是单次任务输入。Agent 的执行边界由 System Prompt 划定。
- [ ] **Few-shot Prompting**: 给模型 2-3 个示例 → 比零样本更稳定。Web3 场景中，Few-shot 常用于格式化输出（如统一 JSON schema）。
- [ ] **Chain-of-Thought（CoT）**: 要求模型"一步一步思考"。在复杂交易分析、合约审计等场景中显著降低错误率。
- [ ] **Tool-use Prompt**: 告诉模型何时调用工具、如何解析工具输出。这是 Agent 从"聊天"进入"执行"的关键跳板。
- [ ] **Prompt Injection 风险**: 用户输入可能覆盖 System Prompt。Web3 场景中尤为危险——不要让用户 Prompt 直接控制钱包调用。

### AI × Web3 关联

- Prompt 是 Agent 的"护栏"——在 Web3 执行场景中，护栏不够 = 资金风险
- Tool-use Prompt 决定了 Agent 能否正确识别"该调用合约"还是"该回答问题"
- 好的 Prompt 在 Web3 场景中 = 防御第一道线

### 挑战实践思路

> 为"链上交易分析助手"写一个 System Prompt，要求：
> 1. 定义角色和输出格式
> 2. 区分"链上事实"和"模型推断"
> 3. 设定安全边界（不主动发起交易）
