## 学习内容

### Prompt 章节核心要点

- [x] **System Prompt vs User Prompt**: System Prompt 定义角色/边界/规则，User Prompt 是单次任务输入。Agent 的执行边界由 System Prompt 划定。
- [x] **Few-shot Prompting**: 给模型 2-3 个示例 → 比零样本更稳定。实践方法：找理想输出 → AI 生成 Prompt → 测试迭代。
- [x] **Chain-of-Thought（CoT）**: 要求模型"一步一步思考"。在复杂交易分析、合约审计等场景中显著降低错误率。
- [x] **Tool-use Prompt**: 告诉模型何时调用工具、如何解析工具输出。这是 Agent 从"聊天"进入"执行"的关键跳板。
- [x] **Prompt Injection 风险**: 用户输入可能覆盖 System Prompt。Prompt 是软约束，真正的边界要由代码/权限/校验承担。
- [x] **Prompt 工程分工**：Prompt 负责"说清楚想干嘛"，Code 负责"不让它干不该干的"。

### Context 章节核心要点

- [x] **Context Window**：窗口大 ≠ 好用，模型会"看见但抓不住重点"
- [x] **Context Engineering**：五层架构 — 指令层/任务层/事实层/知识层/记忆层，每层带标签进入
- [x] **Memory**：过去允许 ≠ 现在允许，涉及身份/权限/资产必须重新授权
- [x] **Knowledge Base**：知识库要标注来源/版本/时效/是否官方

### AI × Web3 关联

- Prompt 是 Agent 的"护栏"——在 Web3 执行场景中，护栏不够 = 资金风险
- Tool-use Prompt 决定了 Agent 能否正确识别"该调用合约"还是"该回答问题"
- 好的 Prompt 在 Web3 场景中 = 防御第一道线
