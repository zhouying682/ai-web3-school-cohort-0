## 学习内容

### LLM 章节核心要点

- [ ] **第一性原理**: LLM 输出是概率合理的，不是天然可信事实。模型 = 推理入口，不是事实源。
- [ ] **Token**: 文本切分基本单位，不等于一个字/单词。代码、JSON、长标识符更吃 token。
- [ ] **Embedding**: 文本→向量，用于搜索/RAG/聚类。语义相似 ≠ 事实正确。
- [ ] **Transformer**: Attention 机制。擅长找模式，不等于有稳定数据库。
- [ ] **Hallucination**: 幻觉不只是质量问题，在执行系统中是流程风险。靠外部校验固定，不靠 prompt 修。
- [ ] **Multimodal**: 多模态让模型读懂截图/图表/控制台，但关键判断仍需结构化数据。

### AI × Web3 定位

```
数据层 → 编排层 → 执行层 → 安全层
  ↑        ↑        ↑        ↑
 RPC    Prompt   钱包调用   Guard/日志
 索引    RAG      合约交互   权限策略
预言机  Agent     签名      人工确认
```

LLM 位在理解和生成层。越靠近执行层，越要把 NL 输出变成可验证对象。

### 最小实践思路

> 输入交易哈希 → 读取链上数据 → LLM 生成解释 → 区分"链上事实"和"模型推断"
